# 产品准则

## 文档与代码注释规范

- **简洁易懂**：强调直观、易于快速阅读。文档和注释应直接切入重点，避免冗余，适合敏捷开发节奏。
- **中文本地化**：所有文档、代码注释及 Git 提交信息均须使用中文。
- **高价值注释**：注释应侧重于解释复杂的逻辑和“为什么”这么做，而不是简单重复代码本身的功能。

## 架构与设计哲学

- **显式优于隐式**：配置、数据流和模型交互应清晰可见。避免使用过于复杂的自动化推断或“魔法”逻辑，确保在分布式环境下系统的行为是可预测且易于调试的。
- **模块化解耦**：坚持 Config, Task, Ops, Pipeline 的分层架构，确保各组件职责单一，易于测试和替换。

## 错误处理与日志记录

- **结构化日志**：系统应采用结构化（如 JSON 格式）记录日志。每条日志应包含必要的上下文信息（如 Task ID, Actor ID, 资源占用情况），以便于在分布式环境下进行高效的故障排查、监控和性能分析。

## 用户交互设计

- **纯后端专注**：当前阶段项目不考虑图形用户界面（UI）。核心交互通过配置文件、Python API 和 CLI 工具完成，重点在于保证接口的稳定性和逻辑的严密性。
