# 初始概念

## 项目目标

DaseR 旨在为数据处理与分析人员提供一个基于大模型的大规模多模态数据处理框架。它利用 Ray 框架的分布式计算能力，结合 vLLM 等高效推理引擎，实现对海量多模态数据（文本、图像等）的链式处理与分析。

核心解决问题：
-   **集成简化**：极大降低 Ray Data 与 vLLM 集成进行大规模批量推理的技术门槛。
-   **架构复用**：提供一套模块化、分层解耦（Config, Task, Ops, Pipeline）的数据处理与推理流水线架构。
-   **资源管理**：高效管理多 GPU 环境下的资源分配、显存控制与任务调度。

## 核心功能

-   **多模态链式处理**：支持将文本、图像等多种模态的数据处理任务（如清洗、提取、推理）串联成灵活的流水线。
-   **性能优化**：针对多模态任务处理进行深度性能优化，包括批处理策略、流水线并行和显存优化。
-   **模块化任务定义**：提供清晰的任务契约和 Prompt 构建机制，便于扩展新的业务逻辑。

## 关键非功能性需求

-   **性能优先**：最大化推理吞吐量和硬件资源利用率。
-   **可扩展性**：架构设计需易于添加新的模型类型、数据算子和任务模版。
-   **易用性**：提供简洁的配置接口和直观的任务定义方式，降低用户上手难度。
-   **轻量化**：专注于核心处理能力，减少不必要的外部依赖，便于快速部署。

## 开发与生态约束

-   **Ray 生态兼容**：确保与 Ray 官方生态组件（如 Ray Serve, Ray Train）保持良好的兼容性。
-   **中文本地化**：
    -   AI 辅助交互使用中文。
    -   所有项目文档（Readme, Spec, Plan 等）使用中文编写。
    -   代码注释必须使用中文。
    -   Git 提交信息（Commit Message）必须使用中文。
